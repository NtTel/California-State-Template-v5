Update your template

  1. Save backup of your current site

    Before updating your version of State Web Template save a backup of your current site. This will ensure a functional site if system errors occurs.
  2. Download the latest State Web Template release

    Use our GitHub repository to download the latest release. If you are using State Web template CDN update your CDN links to the latest release.
  3. Compare changes

    Check our version release update notes above and see if any changes might affect your website. If they do, complete a thorough code comparison. You have several options to compare your files:

    Option 1. Compare files locally against downloaded State Web Template latest release.

    Option 2. Compare your local files with the latest State Web Template code on GitHub.

    Option 3. Compare between specific State Web Template release branches on GitHub.

    To do so you need to make new pull request then can select a release branch name from the compare drop down menu at the top of the page. Here's an example of a comparison between two State Web Template release branches.
  4. Inspect CSS and JavaScript changes

    Most of the changes are usually made in the CSS and JavaScript files. Check and compare those changes thoroughly. You will find the State Web Template CSS and JavaScript source files in /src/scss/ and /src/js/ folders. Update your source CSS and JS files if necessary.

  5. Look for structural changes

    Look for any code changes in your structural components, such as utility-header, site-header, global-footer. You will find those components in State Web Template /src/_includes/ folder. Update your code if necessary.

  6. Look for components and content patterns code changes

    Look for any structural code changes inside of your components or content patterns such as accordion, tabs, cards. You will find those components in State Web Template /src/_includes/modules/ folder. Update your code if necessary.

  7. Copy over the site fonts and icons fonts files

    Copy over the fonts files from /src/fonts/ folder
  8. Build and run your site

    Build and run your site locally. Make sure your site doesn't have NPM or other errors when using build commands. Also, check if there are any errors in your browser's console log.
  9. Test all your pages

    After finishing updating your website do regression testing. Also, make sure you do automatic and manual accessibility testing of all updated pages.
